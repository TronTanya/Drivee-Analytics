# Учётные записи для демонстрации

Пользователи создаются **идемпотентно** при выполнении сида БД:

```bash
make seed
# или из каталога backend:
# python scripts/seed_demo_data.py
```

Логин на форме входа — **email**. Пароль для всех записей ниже один и тот же: **`demo123`**.

| Email | Роль (`role_key`) | Назначение для демо |
|-------|-------------------|---------------------|
| `admin@drivee.local` | admin | Словарь, коррекции, системные сценарии, полный доступ |
| `manager@drivee.local` | manager | Операционный контур, SLA, отмены |
| `marketer@drivee.local` | marketer | Кампании, каналы, воронка |
| `executive@drivee.local` | executive | Обзор KPI, «board-ready» срез |

Все пользователи помечены `is_demo_user=True`, привязаны к workspace **`demo-workspace`** (после сида).

**Важно:** если пользователь уже существовал в БД до смены пароля в сидере, хеш **не обновится** при повторном `make seed` — только отсутствующие строки добавляются. Для гарантированного пароля `demo123` удалите тестового пользователя из БД или выполните сброс пароля через ваш админский поток.

**Старый bootstrap:** ранее в `bootstrap_drivee.sql` создавались адреса `*@drivee.demo` с фиктивным хешем — вход с `*@drivee.local` и `demo123` в такой БД невозможен. Либо пересоздайте том/БД с актуальным bootstrap, либо выполните `make seed` (добавит `*@drivee.local`), либо обновите строки вручную в таблице `users`.

Подробнее о сценарии экрана: [demo-script.md](./demo-script.md).
